The Problem

Sensors Lie — All of Them

Every physical sensor introduces measurement noise. Without correction, small errors accumulate into catastrophic positional drift.

GPS Noise

Typical civilian GPS has ±2–5m horizontal error. At 10m altitude, that's 20–50% positional uncertainty. Multipath, ionospheric delays, and satellite geometry all contribute.

IMU Drift

Accelerometers integrate twice for position — noise becomes quadratic error over time. A 0.1 m/s² bias causes 18m drift in just 60 seconds of dead-reckoning.

Barometer Variance

Pressure-based altitude has ±1–3m accuracy and is heavily affected by wind gusts, temperature gradients, and prop wash from rotors.

The Result

Raw sensor fusion without filtering leads to oscillating control loops, poor hold performance, and in the worst case — a crash. We need an optimal estimator.

Core Concept

What is the Kalman Filter?

An algorithm that produces the statistically optimal estimate of a system's state by combining a mathematical model with noisy measurements.

01

Predict

Use the physical model to predict the next state. Where do we think the drone will be, based on last known position and velocity?

02

Measure

Read the noisy sensor data. GPS reports a position, IMU reports acceleration. These are imperfect observations of reality.

03

Update

Compute a weighted blend of prediction vs. measurement. Trust each proportionally to their uncertainty (covariance).

04

Repeat

Feed the updated estimate back as the new prior. Every cycle the filter converges toward the true state with minimum variance.

Key insight: The filter doesn't just filter — it predicts the future using physics, then corrects with reality. This is what separates it from a simple moving average.

Mathematical Formulation — Phase 2

Update Equations

Kalman Gain — the key weight
Kk = Pk|k-1 HT / (H Pk|k-1 HT + R)
High K = trust the sensor. Low K = trust the model. R is measurement noise covariance.
State Update
xk = xk|k-1 + Kk (zk - H xk|k-1)
New state = Predicted state + Kalman Gain x (Actual measurement - Predicted measurement)
Covariance Update
Pk = (I - Kk H) Pk|k-1
Updated covariance — always smaller than predicted covariance, uncertainty reduces every cycle
The term (z - Hx) is called the innovation — how surprised the filter is by the measurement.
Parameter Intuition

Tuning Q vs R

Q
Process Noise Covariance

Represents uncertainty in your physical model.

High Q = model is unreliable, trust sensor more, filter responds faster but noisier.

Low Q = model is trusted, filter is smooth but slow to react to real changes.

R
Measurement Noise Covariance

Represents how much you trust the sensor.

High R = sensor is noisy, trust model more, output is very smooth but lags reality.

Low R = sensor is trusted, output tracks measurements closely, less filtering effect.

There is no universally "correct" Q/R ratio. Optimal tuning is application-specific and often done empirically via logged flight data.

Filter Variants

Beyond the Basic Kalman

LINEAR KF

Works only for linear systems. Best case scenario. What we simulated today. Foundation of all variants.

OPTIMAL for linear + Gaussian noise
EXTENDED KF (EKF)

Linearizes nonlinear models using Jacobian matrices. Used in ArduPilot/PX4. Handles real drone physics — attitude, position, velocity fusion.

Industry standard for drone EKF3
UNSCENTED KF (UKF)

Uses sigma points to propagate through nonlinearities without computing Jacobians. More accurate than EKF for highly nonlinear systems.

Higher accuracy, higher compute cost
ArduPilot's EKF3 fuses GPS, barometer, magnetometer, IMU and optical flow into a 24-state vector, running at 400Hz on modern flight controllers.
Summary

Key Takeaways

What makes the Kalman filter indispensable for autonomous systems

01

Optimal estimator — minimizes mean squared error for linear Gaussian systems. No other filter does better.

02

Predict + Update loop — uses physics to forecast, then corrects with sensor data. Handles brief sensor dropout gracefully.

03

Q and R are the tuning levers. Balance model trust vs. sensor trust to get the desired filter response.

04

Without Kalman, GPS-based position hold would be impossible. Raw GPS jitter alone would destabilize the control loop.

05

EKF extends this to nonlinear drone dynamics — attitude, velocity, and position are fused simultaneously in 24 states.

06

Recursive by design — no need to store history. Each cycle takes O(n^2) time where n is the state vector dimension.
